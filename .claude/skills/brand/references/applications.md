# Imagery & Cross-Media Applications

## Imagery Rules

Imagery stays monochrome-disciplined — never introduces stray color:

- **Grayscale** — neutral, default for photography.
- **Duotone** — black → white mapped through the image (high-contrast, editorial).
- **Tritone** — black + accent + white, using the accent (or the client's) as the
  midtone for a branded treatment.

Real product screenshots are exempt (they show true UI), but frame them on
neutral surfaces. Decorative imagery follows the duotone/tritone rule.

## Cross-Media Applications

The same tokens drive every surface. Keep the accent rationing and type roles
identical across all of them.

| Surface | Theme | Notes |
|---|---|---|
| Web / app | Full system, equal light/dark | Canonical implementation |
| Presentations (slides) | Dark by default (`bg` #0a0a0a) | Titles `fg`, body `fg-secondary`, one accent element per slide, Bitcount wordmark on title/closing slide only |
| Business cards | Neutral stock (black or white) | Wordmark + mark; accent on the mark only; contact details `fg-secondary`/mono |
| Email | Light theme | Accent on header rule and primary button only; `body-md` Inter; max-width ~600px; never rely on background images |
| Documents (proposals, reports) | Light theme | Inter, generous margins, 65ch measure; accent on section markers and key figures; mono for code/data |
| Social / OG images | Dark theme | Large `display` type, wordmark, one accent element; safe margins for cropping |
| Favicon / avatar | — | Square mark or "HE" monogram — never the full wordmark |
